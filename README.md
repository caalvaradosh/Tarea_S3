# Tarea Semana 3 - Comparación entre Programación Tradicional y Programación Orientada a Objetos en Python

## Estudiante

**Nombre:** Clide Alvarado

## Descripción del proyecto

Este proyecto tiene como objetivo comparar dos enfoques de programación en Python para resolver el mismo problema: gestionar la información básica de una mascota.

Los datos manejados son:

* Nombre
* Especie
* Edad

## Programa Tradicional

El programa tradicional fue desarrollado utilizando variables y funciones. Solicita los datos de una mascota mediante teclado y muestra la información registrada de forma organizada.

Archivo:

* `programacion_tradicional/tradicional.py`

## Programa Orientado a Objetos

El programa orientado a objetos fue desarrollado mediante una clase llamada `Mascota`, que contiene atributos y métodos para representar una mascota y sus comportamientos.

Archivos:

* `programacion_poo/mascota.py`
* `programacion_poo/main.py`

## Reflexión

La Programación Tradicional organiza el código mediante funciones y variables, siendo adecuada para programas pequeños y sencillos. Sin embargo, cuando los proyectos crecen, puede resultar más difícil mantener y reutilizar el código.

Por otro lado, la Programación Orientada a Objetos permite organizar mejor la información mediante clases, objetos, atributos y métodos. Este enfoque facilita la reutilización del código, mejora el mantenimiento de las aplicaciones y permite representar situaciones del mundo real de forma más estructurada.

En esta práctica se pudo observar que ambos enfoques resuelven el mismo problema, pero la Programación Orientada a Objetos ofrece una estructura más organizada y escalable.
